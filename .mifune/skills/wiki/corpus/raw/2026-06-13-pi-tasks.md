## README facts

The README describes pi-tasks as a pi extension for Claude Code-style task tracking and coordination in pi, with structured tasks, dependency management, a persistent visual widget, system-reminder injection, shared file-backed task lists, file locking, background process tracking, and subagent integration.

It exposes seven LLM-callable tools: `TaskCreate`, `TaskList`, `TaskGet`, `TaskUpdate`, `TaskOutput`, `TaskStop`, and `TaskExecute`. The `/tasks` command opens an interactive menu for viewing, creating, clearing, and configuring tasks.

Task storage is controlled by the `/tasks` settings `taskScope`: `memory`, `session` (default, stored at `<cwd>/.pi/tasks/tasks-<sessionId>.json`), or `project` (stored at `<cwd>/.pi/tasks/tasks.json`). Settings are saved to `<cwd>/.pi/tasks-config.json`.

Environment overrides include `PI_TASKS=off` for in-memory-only CI/automation, `PI_TASKS=<name>` for a named shared list at `~/.pi/tasks/<name>.json`, and explicit absolute or relative JSON paths. Named and explicit paths use a file-locked store for multi-session coordination.

`TaskExecute` requires `@tintinweb/pi-subagents`. Tasks must be pending, have `agentType` set, and have completed blockers. Each task spawns an independent background subagent; auto-cascade can execute newly unblocked dependents through the task DAG.
