# `.openharness/`

Host-side configuration mounted into the sandbox container at `/home/sandbox/.openharness/`. Read by the install flow and the container build; the directory is bind-mounted by `.devcontainer/docker-compose.yml`.

## Files

| File | Tracked? | Consumer | Purpose |
|---|---|---|---|
| `config.json` | yes | `install.sh`, `install/banner.sh`, all `docker-compose.*-host.yml` headers | `composeOverrides[]` — list of overlay compose files to apply on top of the base `docker-compose.yml`. Modified by `install.sh` at install time when host UID ≠ 1000. |
| `.gitignore` | yes | git | Excludes per-host runtime artifacts (`Caddyfile`, `exposures.json`). |
| `Caddyfile` | no | (dormant) | Reverse-proxy routes for the optional gateway overlay. Was generated by `oh expose` until US-006 (v0.7) removed the CLI; regenerator is being redesigned for v0.8. |
| `exposures.json` | no | (dormant) | Source-of-truth for `Caddyfile` rendering. Same status as above. |
| `agent/auth.json` | no (gitignored via root `**/auth.json`) | mifune pi pack runtime | Per-user credential symlink. Only present if a pack that needs it is installed. |

## Conventions

- **Hand-edit `config.json`** to opt into / out of overlays. Each overlay's own header documents its requirements.
- **Do not commit** anything in this directory beyond `config.json`, `.gitignore`, and this README.
- **Do not assume** `Caddyfile` / `exposures.json` are kept in sync — there is no regenerator on `main` today.
