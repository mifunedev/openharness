# Rules — Canonical Project Standards

Canonical, tool-agnostic rules for coding, git workflow, process, and harness architecture. Claude Code reaches them via the `.claude/rules` symlink at the repo root; other tools (e.g., Codex, Pi) may consume files directly.

See `repo-layout-source.md` for the single-source-of-truth pattern and `directory-readme.md` for the README convention that documents these files.
