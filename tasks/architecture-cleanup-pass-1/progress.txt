## Codebase Patterns

- Remote-branch prune stories need an explicit deletion set: if the age/PR gates produce zero eligible branches, run only local `git remote prune`/`fetch --prune` and document preserved backup branches with unique commits rather than broadening scope.
- Worktree registrations can point at host-only paths while ignored in-repo directories with `.git` pointer files still exist; if the registered path is unreachable and there is no open PR, treat the local directory plus admin registration as stale cleanup candidates.
- Deleting a script can leave dangling basename-only references even after exact `install/...` path greps pass; grep both the full path and the basename for removed helpers.
- `pnpm -r --if-present run typecheck` is not part of the pre-commit hook but PRD quality gates may still require it; the docs app currently typechecks under TypeScript 5.6 with `ignoreDeprecations: "5.0"`.
- Untracked workspace husks (for example ignored `dist/` + `node_modules/` directories) do not appear in `git status`; verify cleanup with `find`/`du` plus `git ls-files <path>` rather than expecting a staged deletion.
- `git ls-files | xargs grep ...` can surface tracked directory symlink errors from `.codex/` / `.openharness/`; redirect stderr or use `git ls-files -z | xargs -0 grep ... 2>/dev/null` when checking for real content hits.
- When a cleanup acceptance criterion requires a removed filename to have zero tracked references, check task docs too; PRD/prompt text can be the remaining match even after code/docs are clean.
