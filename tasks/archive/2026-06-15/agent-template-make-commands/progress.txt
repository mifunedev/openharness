## Codebase Patterns

- **Real Makefile targets** (source of truth, `Makefile:44`): `sandbox, shell, destroy, stop, logs, ps, restart, config, help`. No `quickstart`, no `list`. `make sandbox` = `docker compose up -d --build` (`Makefile:46-47`).
- **`make shell` takes a positional container arg** (`Makefile:33-40`), NOT `NAME=`. Default container is `$(SANDBOX_NAME)` → `openharness` or `sandbox.name` from `harness.yaml` (`Makefile:15-18`). User override is the env var `SHELL_USER=<user>` (`Makefile:22,49-50`).
- **`format:check` is per-package** (`pnpm -r --if-present run format:check`); the repo-root `.github/ISSUE_TEMPLATE/agent.md` is not inside any workspace package, so it is not reformatted by the formatter — the check passes regardless. Don't expect prettier to touch this file.
- **Per-agent branch/worktree are manual conventions** (`context/rules/git.md` §Worktrees): `git worktree add -b agent/<name> .worktrees/<name> development`. The Metadata `branch`/`worktree_path` fields are intentionally preserved (US-003 guards them); no make target auto-creates them.
- **Story boundaries**: US-001 = sections 1–2 only; US-002 = section 3 (Verify, still has `make list`/`SOUL.md`); US-003 = guard frontmatter + everything above section 1. Stay in your section.
