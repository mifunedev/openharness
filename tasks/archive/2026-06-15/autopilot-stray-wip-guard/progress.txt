## Codebase Patterns

- **Single edited file**: this task touches one skill (`.claude/skills/autopilot/SKILL.md`)
  plus two probes (`evals/probes/{clean-restore,owned-surface-guard}.sh`) and
  `evals/RESULTS.md`. The PRD's Non-Goals forbid touching `crons/autopilot.md`,
  `scripts/cron-runtime.ts`, any other skill, or sandbox app code.
- **`OWNED_PATHS` is load-bearing-unquoted**: it is `.claude/ context/ docs/ scripts/
  crons/ wiki/ evals/ memory/ tasks/ CHANGELOG.md` and MUST be passed bare
  (`-- $OWNED_PATHS`). Quoting (`"$OWNED_PATHS"`) collapses it to one pathspec that
  matches nothing → the clean check passes vacuously. Defined once at the top of the
  §1 "Require clean state" bash block (`grep -c 'OWNED_PATHS=' SKILL.md` must stay 1).
- **§1 self-heal vs MAIN check diverge (US-002+)**: the self-heal stranded-branch
  conditional (`if [ "$BRANCH" != "development" ] && git diff --quiet …`) stays
  tree-wide and is the ONLY permitted tree-wide `git diff --quiet`; the MAIN owned check
  gets scoped. Probes must exempt the self-heal line via BRANCH/rev-parse context.
- **Restore is a two-step (US-004)**: `git checkout development -- $OWNED_PATHS` (scope
  first) THEN `git checkout development` (switch HEAD). After the change exactly 1
  `git checkout -f development` remains (the self-heal). Do NOT use `git restore --source=`
  (it does not switch HEAD — the HIGH critic finding).
- **SHA-accurate bookkeeping = 2 commits/iteration**: commit the implementation first to
  obtain the short SHA, then record `passes:true` + SHA in `prd.json`/`progress.txt` and
  commit that. `Submitted-by:` trailer is mandatory (value from `$RALPH_HARNESS`, e.g.
  `Claude`). Quality gate: `pnpm run lint` works; there is no `type-check` script (it's
  `typecheck`), irrelevant for markdown-only edits.
- **Repros: never `rm -rf .codex` / a tracked dir.** `.codex/` already EXISTS as a tracked
  directory (config.toml, hooks/, agents, …). A foreign-edit repro that does
  `mkdir .codex; … ; rm -rf .codex` DESTROYS the real tracked tree (shows up as a wall of
  ` D .codex/*` in `git status`). Recovery: `git restore --staged .codex && git checkout
  -- .codex`. Use a guaranteed-untracked sentinel path for foreign-edit repros, or better,
  test the predicate logic in isolation without mutating the real tree.
- **Liveness convention** (§ Guidelines "Liveness on every path"): the canonical append is
  `printf '[%s] autopilot: %s\n' "$(date -Iseconds)" "<TOKEN>" >> crons/.cron.log` — guard
  it with a trailing `2>/dev/null || true` so a missing/unwritable log never crashes the
  skill. Reuse this exact shape for any new status token (US-003 did for BLOCKED-OWNED-WIP).
- **`git checkout -f development` is a count-1 invariant (US-004+).** Only the §1 self-heal
  (line ~81) may carry the literal forced form. Restore-site PROSE in §5/§6/§7/Guidelines
  must describe the self-heal WITHOUT writing the exact substring `git checkout -f development`
  (say "its forced tree-wide checkout / the `-f` form") — otherwise the prose mention inflates
  `grep -c 'git checkout -f development'` above 1 and breaks US-006's exactly-1 probe assertion.
- **Grep `$OWNED_PATHS` literally → use `grep -F` (US-006+).** Matching the skill substring
  `git checkout development -- $OWNED_PATHS` with default BRE returns 0: GNU grep treats the
  bare `$` mid-pattern as an end-of-line anchor, so the match silently fails. Use `grep -Fc`
  (fixed-string) or escape as `\$OWNED_PATHS`. The probes MUST use `grep -F` for any
  `$OWNED_PATHS`-bearing literal — US-007's owned-surface-guard probe needs the same. Add
  `# shellcheck disable=SC2016` on the single-quoted line (the literal must NOT expand).
- **Symmetric-oracle test recipe (US-006/US-007).** Verify exit-1-against-unmodified without
  mutating the tree: build a temp ROOT — `git show origin/development:.claude/skills/autopilot/SKILL.md`
  into `$TMP/.claude/skills/autopilot/SKILL.md`, copy the probe into `$TMP/evals/probes/`, and
  run it there (the probe resolves ROOT from its own `$BASH_SOURCE` dir, so it reads the temp
  skill). Exit 0 against the live skill, exit 1 against the temp old skill. SKIPPED (exit 2) is
  reachable by copying the probe alone into an empty temp dir (no skill present).
- **Negative "no unscoped leak" assertion = double `grep -v` (US-007+).** To prove the
  MAIN check has NO tree-wide `git diff --quiet` while exempting the §1 self-heal: list every
  `git diff --quiet` line (`grep -n`), drop the scoped lines (`grep -vF -- '-- $OWNED_PATHS'`),
  then drop the self-heal (`grep -vE 'BRANCH" != "development"|rev-parse --abbrev-ref HEAD'`);
  anything left is a leak. The scoped/self-heal forms share the token `git diff --quiet`, so a
  plain presence grep can't distinguish them — the subtractive double-`-v` does. Guard the
  pipeline with `|| true` (under `pipefail` an empty final grep returns 1).
