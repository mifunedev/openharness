## Codebase Patterns
- `.github/workflows/ci-harness.yml` gates harness changes by `paths:` filters on BOTH `push` and `pull_request`. Any new tracked surface that needs CI coverage must be added to BOTH blocks or it silently merges green.
- hadolint config lives at repo-root `.hadolint.yaml`; the `hadolint/hadolint-action` auto-discovers it (we also pass `config:` explicitly). Validate the ignore list mount-free with `docker run --rm -i hadolint/hadolint hadolint --ignore <CODE>... - < Dockerfile` (host bind-mounts to the docker daemon fail in this sandbox; stdin works).
- shellcheck `-S warning` excludes info/style notes (e.g. the intentional SC2015 `A && B || C` idiom in entrypoint.sh) so the gate is green without `# shellcheck disable` clutter.
- `actionlint` (via `docker run --rm -i rhysd/actionlint -no-color - < workflow.yml`) is the authoritative local validator: YAML + Actions schema + embedded shellcheck on `run:` steps in one pass.
