## Codebase Patterns

- **Extract the live Step C-2 block** (for US-003's probe and for testing): anchor on the
  heading line `**Step C-2 — compare cron file mtimes:**`, then capture the lines between the
  next ` ```bash ` fence and its closing ` ``` `. Working awk:
  `awk '/\*\*Step C-2 — compare cron file mtimes:\*\*/{hit=1;next} hit&&/^```bash$/{cap=1;next} cap&&/^```$/{exit} cap{print}' .claude/skills/drift-check/SKILL.md`
- **Schedulable-cron predicate** mirrors `scripts/cron-runtime.ts` `parseCronFile`+`loadCrons`:
  line-1 `---` (strip trailing `\r`), a closing `---`, an anchored non-comment
  `^[[:space:]]*schedule:`, and not `enabled:` false (bare/quoted). Comment lines (`# schedule:`)
  and substring keys (`pre_schedule:`) are excluded naturally by the `^[[:space:]]*schedule:` anchor.
- The predicate function is invoked as `is_schedulable_cron "$f" || continue`, so under
  `set -e` the `||` context suspends `-e` for the whole function body — intermediate
  `... || return 1` lines are safe.
- Real tree (2026-06-13): schedulable = autopilot, cleanup-tasks, eval-weekly, heartbeat;
  excluded = README.md. `crons/README.md`'s `---` lines are at file lines 15/24 inside a code
  fence — line 1 is an H1, so the line-1 guard excludes it.
- **Behavioral extract-and-run probe** (US-003 pattern, reusable for any "lock a SKILL.md bash
  block" probe): (1) extract the live block, echo it to stderr, REGRESSION if empty or it no
  longer iterates the expected glob; (2) build fixtures in a `mktemp -d` with the cleanup `trap`
  registered BEFORE any fixture; (3) write the block to `$WORK/block.sh` and run it as
  `output="$(cd "$WORK" && RUNTIME_START=1 bash block.sh 2>/dev/null)" || true`; (4) assert on the
  block's *observable stdout contract* (`DRIFT-CHECK (C): <file> ...` lines), never the internal
  array name, so the probe survives a rename of `INERT`. A `DRIFT_CHECK_SKILL=<path>` env override
  (default = live path) makes the REGRESSION/SKIPPED paths testable without editing the real skill.
