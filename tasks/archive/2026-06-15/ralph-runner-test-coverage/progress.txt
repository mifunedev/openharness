## Codebase Patterns

- **Invoke the script under test via `bash`, not `sh`.** `scripts/ralph.sh`
  uses `[[ ]]`, `BASH_SOURCE`, arrays (`POSITIONAL+=`), and `PIPESTATUS` â€”
  none POSIX. `harness-config.test.ts` uses `sh`; ralph must use `bash`.
- **`run(args, opts?)` helper** mirrors `harness-config.test.ts` but extends
  the signature to `run(args: string[], opts?: { env?, cwd? })` via
  `spawnSync("bash", [SCRIPT, ...args], { encoding, env, cwd })`. The
  `env`/`cwd` knobs are needed downstream by US-002's hermeticity setup.
- **`envWithout(...keys)`** returns `{...process.env}` with keys deleted â€”
  used to unmask the `--harness` no-value guard (RALPH_HARNESS is set to
  `claude` in the live Ralph loop env, which would otherwise seed a default).
- **US-001 arg-validation cases all exit before `ralph.sh:358`** (REPO_ROOT
  derivation), so they need no temp cwd or fixture â€” failures fire purely on
  argument shape. Empirically confirmed: `--harness=foo`â†’2/`unknown harness`,
  `--harness`(empty)â†’2/`requires a value`, 0 positionalâ†’2/`Usage:`,
  2 positionalâ†’2/`Usage:`, `Bad_Name`â†’2/`must match`.
- **Hermetic four-file contract (US-002, verified):** spawn the child with
  `{ cwd: tmpDir, env: { ...process.env, REPO_ROOT: tmpDir } }` where `tmpDir`
  is `mkdtempSync(os.tmpdir())`. `ralph.sh:358` re-derives REPO_ROOT via
  `git rev-parse --show-toplevel 2>/dev/null || pwd`; from a non-git tmp cwd the
  rev-parse fails and falls back to `pwd`, redirecting the task-dir lookup
  (:359) to the fixture. The four-file loop (:368-373) exits on the FIRST
  missing file, so omitting exactly one (others present) makes the error name
  that file â€” `expect(stderr).toContain(\`${omitted} is missing\`)` is reliable
  regardless of the omitted file's position in the list. No harness stub: the
  dir check (:361) and four-file check (:369) precede harness resolution (:375).
- **Source guard (US-003) is a `[ "${BASH_SOURCE[0]}" != "$0" ]; return 0`
  block placed between the function definitions and the loop/normal-mode
  body.** Sourcing makes `$0` the parent shell (`bash`) â‰  `BASH_SOURCE[0]`
  (the script path) â†’ early `return 0` (valid in a sourced context). All
  three real paths execute the file (`$0` == `BASH_SOURCE[0]`) so the guard
  is a no-op. **US-004 should source the script in fresh `bash -c 'source
  scripts/ralph.sh; <call>'` subshells** so `set -euo pipefail` is scoped to
  that subshell, not the test runner.
- **Verifying the no-tmux foreground fallback (path 3) needs a PATH with no
  `tmux`.** Build a temp bin, symlink `/usr/bin/*` + `/bin/*` into it, `rm`
  the `tmux` symlink, then run with `PATH=$tmpbin` from a non-git temp cwd
  holding a scaffolded `tasks/<slug>/` whose `progress.txt` already contains
  `STATUS: COMPLETE` (so the loop prints its banner, detects COMPLETE, and
  exits 0 instead of invoking a real harness). `codex` is on the live PATH
  in this env, so a real harness resolves even without a stub.
- **Source-call unit tests (US-004): launch bash by ABSOLUTE path, not bare
  `"bash"`.** `spawnSync` resolves a bare command name against the *child*
  env PATH, so tests that strip PATH to a temp dir (to toggle `codex`
  visibility) would ENOENT on `spawnSync("bash", â€¦)`. Resolve bash once via
  the parent PATH (`spawnSync("bash",["-c","command -v bash"])`) and reuse the
  absolute path. `sourceCall(snippet, {env})` runs
  `spawnSync(BASH,["-c",\`source '${SCRIPT}'\n${snippet}\`],{env})`. The
  US-003 guard returns before the loop body, so only functions load; `set -euo
  pipefail` is scoped to that subshell. The function-under-test as the LAST
  command makes its return value the subshell exit status (read via `status`).
- **PATH control for `codex`-presence tests.** `codex` IS a real binary on the
  live PATH here, so toggle it via `{ ...process.env, PATH: <tmpBin> }`:
  present = a tmpBin holding an executable `codex` stub (`#!/usr/bin/env bash`
  + `exit 0`, `chmodSync 0o755`); absent = an empty tmpBin. `fallback_after_harness`
  and `normalize_harness` use only the `command -v`/`printf`/`case` builtins, so
  a PATH of just the stub dir is enough â€” no coreutils needed. `claude_limit_detected`
  needs real `grep`, so leave its env unset (inherits the parent PATH).
- **No repo-enforced formatter on `scripts/`.** Root `format:check`/`lint`/`typecheck`
  all run `pnpm -r --if-present run â€¦` (workspace packages only) â€” `scripts/__tests__`
  is covered by `pnpm test:scripts` (vitest, no type-check) alone. A bare
  `npx prettier --check` installs default prettier and flags the COMMITTED
  baseline (`harness-config.test.ts`) too, so its warnings are not a gate.
  Verify strict types directly: `npx tsc --noEmit --strict --skipLibCheck
  --module esnext --target es2022 --moduleResolution bundler --typeRoots
