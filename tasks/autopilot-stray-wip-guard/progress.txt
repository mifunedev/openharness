## Codebase Patterns

- **Single edited file**: this task touches one skill (`.claude/skills/autopilot/SKILL.md`)
  plus two probes (`evals/probes/{clean-restore,owned-surface-guard}.sh`) and
  `evals/RESULTS.md`. The PRD's Non-Goals forbid touching `crons/autopilot.md`,
  `scripts/cron-runtime.ts`, any other skill, or sandbox app code.
- **`OWNED_PATHS` is load-bearing-unquoted**: it is `.claude/ context/ docs/ scripts/
  crons/ wiki/ evals/ memory/ tasks/ CHANGELOG.md` and MUST be passed bare
  (`-- $OWNED_PATHS`). Quoting (`"$OWNED_PATHS"`) collapses it to one pathspec that
  matches nothing → the clean check passes vacuously. Defined once at the top of the
  §1 "Require clean state" bash block (`grep -c 'OWNED_PATHS=' SKILL.md` must stay 1).
- **§1 self-heal vs MAIN check diverge (US-002+)**: the self-heal stranded-branch
  conditional (`if [ "$BRANCH" != "development" ] && git diff --quiet …`) stays
  tree-wide and is the ONLY permitted tree-wide `git diff --quiet`; the MAIN owned check
  gets scoped. Probes must exempt the self-heal line via BRANCH/rev-parse context.
- **Restore is a two-step (US-004)**: `git checkout development -- $OWNED_PATHS` (scope
  first) THEN `git checkout development` (switch HEAD). After the change exactly 1
  `git checkout -f development` remains (the self-heal). Do NOT use `git restore --source=`
  (it does not switch HEAD — the HIGH critic finding).
- **SHA-accurate bookkeeping = 2 commits/iteration**: commit the implementation first to
  obtain the short SHA, then record `passes:true` + SHA in `prd.json`/`progress.txt` and
  commit that. `Submitted-by:` trailer is mandatory (value from `$RALPH_HARNESS`, e.g.
  `Claude`). Quality gate: `pnpm run lint` works; there is no `type-check` script (it's
  `typecheck`), irrelevant for markdown-only edits.
