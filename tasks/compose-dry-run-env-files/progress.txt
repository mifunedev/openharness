# Progress — compose-dry-run-env-files

## Codebase Patterns

- `scripts/docker-compose.sh --print-argv` is diagnostic-only; tests that need to prove real lifecycle behavior should use a fake `docker` executable and run the helper without `--print-argv`.
- Read-like compose helper paths may use temporary env files and avoid `exec` so cleanup traps run; lifecycle paths keep `exec docker compose` with the persistent generated env file.
