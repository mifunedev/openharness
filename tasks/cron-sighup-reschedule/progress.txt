## Codebase Patterns

- **No tsconfig at root.** `scripts/*.ts` is transpiled by vitest/esbuild (type
  stripping, no type-check) and editor TS. The root `pnpm run typecheck` only
  covers `packages/**` + `packages/oh`. `@types/node` is NOT installed, so an
  isolated `tsc` on `scripts/cron-runtime.ts` fails to resolve `node:*` imports.
  To genuinely type-check a snippet, drop a temp `.ts` under `scripts/`, run the
  local `./node_modules/.bin/tsc --noEmit --strict --skipLibCheck
  --moduleResolution bundler --module esnext --target es2022 <file>` (avoid
  `--types node`), then delete it. The real CI gate for scripts is
  `pnpm run test:scripts` (vitest, 14 files / 186 tests).
- **`Cron | void` narrows cleanly.** `if (handle) activeJobs.push(handle)` where
  `handle: Cron | void` narrows to `Cron` under `--strict` — no cast needed.
- **Test-injection seams convention.** `loadCrons(dir, logFn=log)`,
  `onJobError(id, err, logFn=log)`, and now `scheduleAll(dir, logFn, mkCron)`
  expose optional last params that default to module-private impls purely so
  tests can drive behavior without real side effects. New seams (e.g.
  `resetActiveJobs()`) should carry a "no external-stability guarantee" comment
  to match the existing ones.
- **Logging is `log(id, status, msg)`** → one tab-separated line to
  `crons/.cron.log` via `appendFileSync`. Tests mock only `appendFileSync`
  (vi.mock("node:fs") pass-through) to observe status tokens. Reuse the private
  `log()` helper for any new token (US-003 RELOAD) — never inline appendFileSync.
- **`scheduleAll` counting is disjoint:** `loadSkips` (invalid at load) +
  `constructSkips` (throws at `mkCron`); `scheduled++` fires on every successful
  `mkCron` regardless of handle truthiness, so a `() => void` test spy still
  counts as scheduled.
