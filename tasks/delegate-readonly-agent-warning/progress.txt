## Codebase Patterns

- `.claude/skills/delegate/SKILL.md` "Worker configuration:" block lives under **§5 Execute waves → sub-step (a)** (around the Model / run_in_background bullets). Anchor edits by that heading text, not line number.
- The §Reference "Key Resources" table is a **2-column** `| Resource | Path |` table near the end of the file. The `Agent: Council` row is a known-stale reference — leave it untouched (out of scope).
- Repo quality gates: `pnpm run lint` and `pnpm run format:check` both delegate to `pnpm -r --if-present` and pass cleanly for docs/skill edits.
- Probe region-scoping (US-003): extract the two warning regions with awk and check their union — `awk '/^Worker configuration:/{f=1} f{print} f && /^[[:space:]]*$/{exit}'` for the bullet block, `awk '/^### Key Resources/{f=1; next} f && /^(### |## )/{exit} f{print}'` for the table. A whole-file grep would over-pass (Critic A finding). Distinguish empty-region (region-not-found) from non-empty-but-missing-phrase REGRESSIONs.
- Non-destructive probe spot-check pattern: build a throwaway `mktemp -d` ROOT (`$TMP/evals/probes/`, `$TMP/.claude/skills/delegate/`), copy the probe in, and `sed` a stripped SKILL.md into place. The probe's `ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"` resolves to `$TMP`, so all four exit states (0/1/1/2) can be exercised without ever dirtying the real tree.
- The eval runner (`.claude/skills/eval/run.sh --probe <id>`) rewrites `evals/RESULTS.md` even for a single-probe run (carries prior rows for the rest). When running it from a non-US-004 story, `git checkout -- evals/RESULTS.md` afterward to keep the commit scoped — US-004 owns that artifact.
