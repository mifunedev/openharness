## Codebase Patterns

- **`run.sh` ROOT derivation is location-sensitive.** `ROOT="$(cd "$SCRIPT_DIR/../../.." && pwd)"` resolves the repo root from the script's own path (`.claude/skills/eval/`). When testing a *copy* of run.sh, it MUST sit at that same relative depth or `$RESULTS` resolves to garbage (`//evals/RESULTS.md`). To compare OLD vs NEW, swap the file in the canonical path — don't drop the old copy at the scratch root.
- **Real probes are non-deterministic across back-to-back runs.** They reach the live repo via cwd and mutate shared state, so two sequential full runs can yield different statuses. To test the *runner's* file-writing behavior (structural equivalence), replace probes with trivial deterministic stubs that `exit 0|1|2|7` — hold probe statuses constant so any diff is attributable to the runner.
- **Mask the timestamp column when diffing RESULTS.md.** `sed -E 's/\| [0-9]{4}-[0-9]{2}-[0-9]{2} [0-9]{2}:[0-9]{2} \|/| TS |/'` — the per-run `last-run (UTC)` column is the only legitimate diff between two structurally-equivalent scoreboards. `(not run)` rows use `| — |` (no date) and are unaffected.
- **`eval-runner-exit.sh` asserts the exit-1 gate via `tail -n 12` + regex `regressions\[@\].*-gt 0.*exit 1`.** Any run.sh refactor must keep `if [ "${#regressions[@]}" -gt 0 ]; then exit 1; fi` within the last 12 lines, or update that probe in the same change.
- **The `--ablate` path `exec`s and replaces the shell (run.sh:~55).** EXIT traps set before it never fire on that path; `tmp=""` makes the guarded trap a no-op there regardless.
