## Codebase Patterns

- `.gitignore` ignores daily-log dirs via `memory/[0-9]*/` (line 44). Only
  `memory/MEMORY.md`, `memory/README.md`, and topic notes persist in git;
  daily `memory/YYYY-MM-DD/log.md` is a local-only working journal.
- Two pre-ignore logs (`memory/2026-05-03/log.md`, `memory/2026-05-04/log.md`)
  remain tracked as a historical vestige — `git ls-files memory/` confirms they
  are the ONLY tracked daily dirs. Do NOT remove them (PRD Non-Goal).
- `context/rules/memory.md` IS `.claude/rules/memory.md` via symlink — edit the
  canonical `context/rules/` path; both views update.
- Quality gate trio: `pnpm typecheck`, `pnpm lint`, `pnpm test:scripts` (196
  tests, ~3s) all green on this branch. Shell is zsh — `${PIPESTATUS[0]}` does
  not capture as in bash; judge pass/fail by the command's own output.
- Commit trailer is mandatory: `Submitted-by: Claude` (from `$RALPH_HARNESS`).
- Probe convention (`evals/probes/*.sh`): mirror `boot-lint-glob.sh` exactly —
  `#!/usr/bin/env bash`, `# tier:` / `# source:` / `# desc:` header trio,
  `set -euo pipefail`, `ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"`,
  3-state oracle (0 PASS / 1 REGRESSION / 2 SKIPPED), all human-readable lines to
  stderr. Grep a missing file silently false-PASSes a "must NOT contain" assertion
  (`grep -q` exits non-zero) — guard with an explicit `[[ ! -f ]]` → exit 2 SKIPPED
  per input file. Use `grep -qF` for literal patterns (e.g. `memory/[0-9]*/`,
  `tracked inside`); strip comment lines with `grep -v '^[[:space:]]*#'` before a
  "rule present" assertion so a commented-out line can't false-PASS.
- Do NOT add `evals/probes/` to the CI boot-lint shellcheck glob — that flips the
  `boot-lint-glob` probe to REGRESSION (it asserts the glob covers exactly the four
  boot-script dirs). Run `shellcheck -S warning` on new probes manually instead.
