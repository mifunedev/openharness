## Codebase Patterns

- **Worktree placement**: V0 work lives at `.worktrees/project/mifunedev/skills/` per `context/rules/git.md` ¬ß Worktrees convention for independent project clones. The root `.gitignore` includes `!.worktrees/project/**` so the subtree is trackable on the harness branch.
- **Branch ref for install.sh**: the existing `mifunedev/skills` GitHub remote uses `master` (not `main`) as its only branch. `install.sh` therefore defaults `REGISTRY_BRANCH=master` and accepts a `MIFUNE_REGISTRY_BRANCH` env override.
- **Checksum algorithm** (DRY across install.sh, validate.sh, refresh-checksums.sh): `find skills/<name> -type f -not -path '*/.*' | LC_ALL=C sort | xargs sha256sum | sha256sum | cut -d' ' -f1`. Canonical doc at `docs/checksum.md`.
- **skills-ref invocation** (pinned per Wave 0 pre-flight): `npx skills-ref@0.1.5 validate <path>`. Recorded in `tasks/mifune-skills-library/verification.md` ¬ß skills-ref invocation.
- **/skill-lint external-folder recipe** (per Wave 0 pre-flight): `/skill-lint` is a pure-LLM skill with no `--path` flag; the V0 recipe is stage-to-`.claude/skills/<name>/`, invoke `/skill-lint`, then `rm -rf` the staging path. End-to-end run is gated on Claude Code session, NOT subagent.
- **Frontmatter portability deny-list** (stricter than upstream spec): `disable-model-invocation`, `user-invocable`, `paths`, `context`, `agent`, `argument-hint`, `arguments`, `hooks` are forbidden at top level. Lift to `metadata.mifune.claude-code.*` if needed. Rationale: `docs/portability.md`.
