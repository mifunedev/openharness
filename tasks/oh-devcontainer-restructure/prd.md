# PRD — RFC #531 Phase 2 (slice 2): relocate harness build assets to `.oh/devcontainer/`

> RFC: https://github.com/mifunedev/openharness/issues/531 — *"Make OpenHarness portable
> via project-local `.oh` infrastructure."* Philosophy: **OpenHarness equips a repository;
> it should not become the repository.**

## Summary

Move the harness's **own** build/bootstrap assets out of the top-level `.devcontainer/`
directory and under `.oh/devcontainer/`, so the boot machinery lives inside the portable
`.oh/` control plane rather than masquerading as repository-level files. The root
`.devcontainer/` is reduced to a thin **compatibility layer** (the VS Code
`devcontainer.json` + the user-owned `.env`/`.example.env`/`.harness.yaml.env`), and that
`devcontainer.json` is **generated** from a canonical source under `.oh/devcontainer/` via a
small `sync` script rather than treated as canonical.

This is the RFC's "Devcontainer" bullet for Phase 2. It **does** touch the running sandbox's
boot path — that is its nature — but it is a pure relocation + reference-repoint with **no
behavior change**: the same image builds from the same context, the same entrypoint runs, the
same compose service comes up. CI's **"Validate sandbox compose and image build"** workflow
(`sandbox-boot-guard.yml`, which `docker compose config` + `docker build` + boots the image
through the healthcheck) is the hard oracle.

## Destination decision (operator-confirmed)

Build assets land at **`.oh/devcontainer/`** — NOT `.oh/templates/.devcontainer/`. Rationale
(operator-selected 2026-06-27):

- `.oh/templates/` stays **purely the downstream scaffold** that `oh init` writes into user
  repos (slice 1 / PR #334, untouched here — zero merge collision on `devcontainer.json`).
- `.oh/devcontainer/` is the **harness's own canonical build context**.
- Root `.devcontainer/` = generated compat layer + user secrets.
- The RFC's literal tree (`.oh/templates/.devcontainer/Dockerfile`) is reinterpreted because
  taking it literally would (a) collide with slice 1's stub and (b) make `oh init` copy the
  full 13 KB Dockerfile + 20 KB entrypoint into every user repo — exactly what the RFC's
  "equip, don't become the repository" philosophy forbids. We honor the philosophy over the
  literal path.

## The wiring (exact path edits — verified against the live tree)

1. **`.oh/devcontainer/Dockerfile`** — build context stays repo root, so only one COPY changes:
   `COPY .devcontainer/entrypoint.sh …` → `COPY .oh/devcontainer/entrypoint.sh …`. All other
   COPY sources (`.oh/cli/`, `.oh/install/`, `workspace/`) are root-relative and unchanged. The
   image-internal `RUN … > /.devcontainer/devcontainer.json` line is an **in-image** path — leave it.
2. **`.oh/devcontainer/docker-compose.yml`** — relative paths resolve from the compose file's
   new dir (`.oh/devcontainer/`), so: `context: ..` → `context: ../..`; `dockerfile:
   .devcontainer/Dockerfile` → `dockerfile: .oh/devcontainer/Dockerfile`; the bind
   `- ..:/home/sandbox/harness` → `- ../..:/home/sandbox/harness`. The in-container healthcheck
   path `/home/sandbox/harness/.oh/scripts/sandbox-healthcheck.sh` is unchanged.
3. **`.oh/devcontainer/docker-compose.hermes-dashboard.yml`** — ports/env only, **no relative
   paths** → moves with zero internal edits.
4. **`.oh/devcontainer/entrypoint.sh`** — its only path ref is `$HARNESS/.devcontainer/.env`
   (user env, stays at root) → **no functional edit**. `client-slack-supervise.sh` /
   `seed-msg-bridge.sh` likewise reference only the root `.env`/`.pi/` → no functional edits
   (fix the stale `# Launched by .devcontainer/entrypoint.sh` comment for accuracy).
5. **`.dockerignore`** — excludes `.devcontainer/` then re-includes `!.devcontainer/entrypoint.sh`
   + `!.devcontainer/Dockerfile`. After the move those negations are **stale** (remove them);
   `.oh/` is not excluded so `.oh/devcontainer/entrypoint.sh` is already in the build context and
   the Dockerfile is read via `--file`.
6. **`.oh/scripts/docker-compose.sh`** — compose `-f` paths `.devcontainer/docker-compose.yml`
   + `.devcontainer/docker-compose.hermes-dashboard.yml` → `.oh/devcontainer/…`. Env-file paths
   (`.devcontainer/.env`, `.devcontainer/.harness.yaml.env`) **stay**.
7. **`.oh/scripts/gateway.sh`** — `seed-msg-bridge.sh` + `client-slack-supervise.sh` exec paths
   → `.oh/devcontainer/…`. `SLACK_ENV="$HARNESS/.devcontainer/.env"` **stays**.
8. **`.oh/scripts/install.sh`** — the harness-repo **detection** test
   `[ -f "$REPO_CANDIDATE/.devcontainer/docker-compose.yml" ]` → `.oh/devcontainer/docker-compose.yml`.
   The `.env` writing paths **stay**.
9. **`.oh/scripts/check-pnpm-pin.sh`** — `DOCKERFILE=…/.devcontainer/Dockerfile` →
   `.oh/devcontainer/Dockerfile` (+ the advice message).
10. **`.oh/scripts/sync-devcontainer.sh`** (NEW) — generator: writes root
    `.devcontainer/devcontainer.json` (the compat layer) pointing at
    `../.oh/devcontainer/docker-compose.yml`; idempotent; `--check` mode exits non-zero on drift.
11. **CI `.github/workflows/ci-harness.yml`** — shellcheck glob `.devcontainer/*.sh` →
    `.oh/devcontainer/*.sh` (root `.devcontainer/` has no `.sh` after the move, so the old glob
    would match nothing and error); hadolint `dockerfile: .devcontainer/Dockerfile` →
    `.oh/devcontainer/Dockerfile`.
12. **CI `.github/workflows/release.yml`** — same shellcheck-glob + hadolint changes, plus
    `docker build -f .devcontainer/Dockerfile` → `.oh/devcontainer/Dockerfile`.
13. **CI `.github/workflows/sandbox-boot-guard.yml`** — `docker build --file
    .devcontainer/Dockerfile` → `.oh/devcontainer/Dockerfile`. Path filters: `.oh/**` already
    covers the new location; **keep** `.devcontainer/**` (compat layer + user env still live there).
14. **`.hadolint.yaml`** — header comment `gates .devcontainer/Dockerfile` → `.oh/devcontainer/Dockerfile`.
15. **`evals/probes/boot-lint-glob.sh`** — the asserted boot-dir list `.devcontainer/ .oh/install/
    .oh/scripts/` → `.oh/devcontainer/ .oh/install/ .oh/scripts/` (+ PASS message). This probe
    guards BOTH `ci-harness.yml` and `release.yml`, so #11/#12 must add `.oh/devcontainer/*.sh`.
16. **`evals/probes/sandbox-boot-guard-ci.sh`** — `has '--file .devcontainer/Dockerfile'` →
    `has '--file .oh/devcontainer/Dockerfile'`. The `.devcontainer/**` path-filter assertion
    **stays valid** (root compat layer remains). Confirm the new `--file .oh/devcontainer/Dockerfile`
    string does NOT trip the workflow's anti-registry regex (it has no `host.tld/…:tag` shape — verified).
17. **New probe `evals/probes/oh-devcontainer-restructure.sh`** — 3-state oracle guarding the move.
18. **Docs** — `.oh/README.md` (devcontainer-location note), root `AGENTS.md` (the one
    `.devcontainer/client-slack-supervise.sh` path ref → `.oh/devcontainer/…`, surgical),
    `docs/integrations/slack.md` (same path ref), `CHANGELOG.md` ([Unreleased] entry),
    `evals/RESULTS.md` (one row).

## Non-Goals (explicit — keep the slice safe and reviewable)

- **No runtime-path change.** The bind mount stays `…:/home/sandbox/harness` and
  `workspaceFolder` stays `/home/sandbox/harness`. Switching to `/home/sandbox/project` is a
  separate Phase-1/Phase-3 concern and a higher-risk change; out of scope here.
- **No `oh init` / `.oh/templates/` changes.** Slice 1 (PR #334) is untouched; the downstream
  scaffold and the harness's own build context are deliberately distinct trees.
- **No behavior change to the image or boot sequence.** Pure relocation + reference repoint.
- **No `oh update` / vendoring** (Phase 3).
- **No move of `.oh/install/`** — it already lives under `.oh/` (the Dockerfile already COPYs
  from `.oh/install/`); only the `.devcontainer/`-resident assets relocate.

## Decomposition — four disjoint-file user stories (one parallel `/delegate` wave)

Verified disjoint (no file owned by two stories), so they run as **one EDIT-only `/delegate`
wave**; the Advisor (the ralph iteration) stages with git, runs the gates, and commits per story.

- **US-001 — Relocate the build assets + fix in-file refs + `.dockerignore`.** Owns
  `.oh/devcontainer/**` (the 6 moved files) and `.dockerignore`.
- **US-002 — Repoint `.oh/scripts/*` + generate the root compat layer.** Owns
  `.oh/scripts/docker-compose.sh`, `gateway.sh`, `install.sh`, `check-pnpm-pin.sh`, NEW
  `.oh/scripts/sync-devcontainer.sh`, and root `.devcontainer/devcontainer.json` (produced by
  running the generator).
- **US-003 — Repoint CI workflows + existing boot probes.** Owns
  `.github/workflows/{ci-harness,sandbox-boot-guard,release}.yml`, `.hadolint.yaml`,
  `evals/probes/boot-lint-glob.sh`, `evals/probes/sandbox-boot-guard-ci.sh`.
- **US-004 — New guard probe + docs + changelog.** Owns NEW
  `evals/probes/oh-devcontainer-restructure.sh`, `evals/RESULTS.md`, `CHANGELOG.md`,
  `.oh/README.md`, root `AGENTS.md` (surgical), `docs/integrations/slack.md`.
