## Codebase Patterns

- **Running gates in a fresh worktree (no node_modules):** this worktree sits *inside* the
  main checkout (`/home/sandbox/harness/.worktrees/...`), so Node module resolution walks up
  and finds the parent's `node_modules` — `vitest` and `fast-check` resolve for free. Run
  vitest via the parent binary: `/home/sandbox/harness/node_modules/.bin/vitest run .oh/cli`
  (scope to `.oh/cli` to cover only the touched tests). `typescript`/`esbuild` are NOT in the
  parent tree (`.oh/cli` is npm-standalone, not a pnpm workspace member), so install them with
  `npm --prefix .oh/cli install` before `typecheck`/`build`.
- **`.oh/cli` is npm-standalone**, not a pnpm workspace member (only `.oh/docs` is). Gate it
  directly with `npm --prefix .oh/cli run typecheck|build`, never via the root pnpm filters
  (the `pnpm --filter './packages/**'`/`'./.oh/**'` trap). `npm install` there is lightweight
  and runs no audit hook (unlike root `pnpm install`'s `pnpm:devPreinstall` audit).
- **NodeNext ESM** (`"type":"module"`, `module/moduleResolution: NodeNext`): local imports of
  `.ts` files MUST use the `.js` extension (`../commands/init.js`). esbuild `bundle:true,
  format:esm` keeps `import.meta.url` a live ref to the *output* file, so module-relative path
  arithmetic (`../../templates` from `dist/oh.js`) holds at runtime.
- **Template-path depth differs by caller location:** `cli.ts`/`dist/oh.js` are 2 levels under
  `.oh/cli` → `../../templates`; the test under `src/__tests__/` is 3 levels up from `.oh/` →
  `../../../templates`. Copying the cli.ts 2-level path into the test silently resolves the
  wrong (absent) dir.
- **Eval probes are 3-state oracles** (0 PASS / 1 REGRESSION / 2 SKIPPED), self-resolve root
  via `ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"`, header `# tier:`/`# source:`/
  `# desc:` lines, every outcome a one-line stderr reason. A feature-gating SKIP keeps a probe
  green on branches without the feature. shellcheck them by hand (`evals/probes/*.sh` is not in
  CI's boot-lint glob).
- **Sandbox secret-guard** denies any Bash command containing the literal `.devcontainer/.env`
  (pattern-matched as an env path) — verify such files via `grep -n`/`node`, not by echoing the
  literal in a shell command.
