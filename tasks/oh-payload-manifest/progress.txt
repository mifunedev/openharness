# Progress — oh-payload-manifest (RFC #531 follow-on, value-first step 3)

## Codebase Patterns

- This slice STACKS on PR #336 (`feat/531-oh-update`); the base already has
  `.oh/cli/src/commands/update.ts` with `runUpdate` + the `assertDestInTarget` path-escape
  guard + a `walkFiles` overlay that ships ALL of `.oh/` minus per-segment node_modules/dist
  (lines ~158-202). This slice EXTENDS that overlay with a manifest filter — it does NOT
  rewrite update.ts.
- The overlay relpaths are POSIX, relative to `.oh/` (`path.relative(fromOh, abs)` at
  update.ts:51). The manifest globs match THAT shape (`cli/**` → `.oh/cli/...`). Because the
  root is `<from>/.oh` and `assertDestInTarget` refuses any dest outside `<target>/.oh`, the
  manifest STRUCTURALLY cannot reach outside `.oh/` — the safety invariant is preserved free.
- The matcher (`lib/manifest.ts`) is zero-dependency by design: the CLI uses only node:fs +
  node:path (no glob lib). `globToRegExp` must detect the 2-char `**` before the 1-char `*`
  and special-case leading `**/` → `(?:.*/)?` so `**/node_modules/**` matches a top-level
  `node_modules/x` too. A no-wildcard pattern is an EXACT match.
- Tests import SOURCE via `.js` specifiers (NodeNext maps `.js` → `.ts`; vitest transforms
  TS). `vitest.config.ts` auto-discovers `.oh/cli/**/__tests__/**/*.test.ts`. Verify with
  `/home/sandbox/harness/node_modules/.bin/vitest run --root <worktree>` (worktree has no
  node_modules; resolution walks up to repo-root node_modules).
- Eval probes are 3-state oracles (0 PASS / 1 REGRESSION / 2 SKIPPED) with `# tier:` /
  `# source:` / `# desc:` headers and `ROOT` via `${BASH_SOURCE[0]}`. `evals/RESULTS.md` is a
  `| probe | tier | last-run (UTC) | status | source |` table, one row per probe id, kept
  alphabetical. The "Eval Probe Regression Gate" CI job fails only on a NEW green→red regression.
- The de-hardcode landmine is LARGELY ALREADY MITIGATED (corrected against the actual #336
  base): `prompt-miner-caps.sh` is NOT under `.oh/` — it lives at
  `.mifune/skills/prompt-miner/prompt-miner-caps.sh`, so `oh update` (which writes only under
  `.oh/`) never ships it. The shipped `.oh/` scripts that DO reference the upstream repo are
  `.oh/scripts/install.sh` (clone target, `${OH_GITHUB_REPO:-mifunedev/openharness}` override +
  a help-text + a `!=` fork-detection comparison) and `.oh/scripts/maintenance/restart-openharness-tmux.sh`
  (`${OH_REPO:-ryaneggz/openharness}` override). Both use the overridable `${VAR:-…}` form. The
  root `crons/*.md` are instance-layer (NOT shipped). So this slice's de-hardcode work is just a
  WHITELIST GUARD PROBE (`:-`/`#`/`!=`/`default:` allowed) locking that no shipped `.oh/` script
  REGRESSES into a bare hardcode — not a behavior change to the live autopilot.

`node_modules/x`. Empty-include→null is the hollow-out guard.
