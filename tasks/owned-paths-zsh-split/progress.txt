## Codebase Patterns

- **OWNED_PATHS fix form (canonical):** native shell ARRAY
  `OWNED_PATHS=(.claude/ context/ docs/ scripts/ crons/ wiki/ evals/ memory/ tasks/ CHANGELOG.md)`
  expanded EVERYWHERE as `"${OWNED_PATHS[@]}"`. Correct under BOTH bash and zsh
  (verified: `set -- "${OWNED_PATHS[@]}"; echo $#` → 10 under each). Never the bare
  `$OWNED_PATHS` (collapses to 1 pathspec under zsh — no SH_WORD_SPLIT) nor quoted
  `"$OWNED_PATHS"` (collapses everywhere).
- **Probe literal matching is `grep -F` (load-bearing):** `${OWNED_PATHS[@]}` contains
  `[` `]` `@`, which are BRE metacharacters. Every literal pathspec assertion uses
  `grep -Fq` / `grep -Fc` / `grep -vF`. A non-`-F` grep silently mis-matches.
- **Substring disjointness:** `$OWNED_PATHS` is NOT a substring of `${OWNED_PATHS[@]}`,
  so the four bare-form absence greps and the array-form presence greps are unambiguous
  and never alias each other.
- **`--cached` is a DISTINCT string:** `git diff --cached --quiet -- $OWNED_PATHS` must
  be replaced AND guarded separately from the plain `git diff --quiet -- $OWNED_PATHS`.
- **zsh-fidelity check pattern:** anchored extraction `grep -oP '^OWNED_PATHS=\(\K[^)]+'`
  (only the real declaration starts at col 0; comment mentions start with `#`), then
  `zsh -c "OWNED_PATHS=($decl); set -- \"\${OWNED_PATHS[@]}\"; echo \$#"` capturing the
  count EXPLICITLY (`|| echo 0`) rather than relying on `set -e` propagation, behind a
  `command -v zsh` skip-guard (exit 2 SKIPPED if absent).
- **Verify gates with error-injection:** copy SKILL.md to a temp ROOT, re-inject each
  broken form, and confirm the probe exits 1 with the named REGRESSION — proves the
  guard is not vacuous (per MEMORY.md pnpm-filter lesson).
