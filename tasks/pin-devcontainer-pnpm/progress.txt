## Codebase Patterns
- The pnpm version source of truth is `package.json` `packageManager` (`pnpm@10.33.0`); the devcontainer Dockerfile must mirror it. `scripts/check-pnpm-pin.sh` enforces the invariant and is run by CI boot-lint + release validate.
- `scripts/__tests__/*.test.ts` is the vitest convention for orchestrator scripts; mirror `harness-config.test.ts` (spawnSync + temp fixtures). Run via `pnpm test:scripts`.
- `ci-harness.yml` boot-lint shellcheck step already globs `scripts/*.sh`, so any new `scripts/*.sh` is auto-shellchecked — don't add a redundant shellcheck step.
