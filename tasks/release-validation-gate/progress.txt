## Codebase Patterns

- GitHub Actions jobs run on separate runners and do not share state, so the
  publish job keeps its own Setup Node + Install pnpm steps even though the
  `validate` job also runs them — this duplication is intentional, not a smell.
- `.claude/rules/` is a symlink to `context/rules/`; edit the canonical
  `context/rules/<file>` and the `.claude/rules/<file>` path resolves to the
  same inode (verify with `diff` exit 0).
- No host PyYAML; validate workflow YAML with the repo's bundled `js-yaml`
  (`node -e "require('js-yaml').load(...)"`).
