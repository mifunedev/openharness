# progress

## Codebase Patterns

- **FR-6 allowlist is enforced by convention, not tooling.** Stick strictly to the file targets the chosen story names. Other paths (especially `context/rules/`, `.claude/`, `crons/`) are off-limits even if they would simplify the work.
- **Topic memory notes are flat under `memory/<topic>.md`** (per `context/rules/memory.md`), not under daily-log subdirs. `memory/x-campaign.md` is the canonical style anchor — terse, link-heavy, factual, no first-person narrative.
- **Pre-commit hook runs `pnpm lint` + `pnpm test`** on every commit. Markdown-only changes pass cleanly; do not use `--no-verify`.
- **Stage files explicitly by path** (`git add <file>`), never `git add -A` — the harness has many concurrent moving parts and broad staging risks pulling in unrelated changes (e.g. `.worktrees/`, daily memory logs, devcontainer state).
- **`prd.json` is the only metadata file you touch per iteration.** Update `passes` and `notes` for the completed story; do not edit other stories' fields.
- **Per-iteration metadata edits accumulate uncommitted.** Per the loop's "stage only files touched by this story" rule, neither `prd.json` nor `progress.txt` are committed — they live as workspace state across iterations. Don't be alarmed by their presence in `git status` at iteration start.
- **CHANGELOG `[Unreleased]` may carry multiple `### Added` subheadings** due to drift; new entries go at the **top** of the topmost `### Added` block to preserve newest-first ordering. Issue-link format per `context/rules/git.md` § Changelog: `([#N](https://github.com/ryaneggz/open-harness/issues/N))`.
