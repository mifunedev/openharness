# progress

## Codebase Patterns
- The repo has TWO TypeScript install regimes: `packages/docs` is a pnpm
  workspace member (in `pnpm-workspace.yaml`); `packages/oh` is an
  npm-standalone package (own `package-lock.json`, built in the Dockerfile,
  NOT in the pnpm workspace). `pnpm --filter './packages/**'` reaches ONLY
  workspace members — it silently skips `oh`. To gate `oh` in CI you must run
  its own `npm --prefix packages/oh ci` + `npm ... run typecheck`.
- A local YAML validator is available transitively at
  `node_modules/.pnpm/js-yaml@4.1.1/node_modules/js-yaml` (not a root dep).
